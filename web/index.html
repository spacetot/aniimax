How It Works

1. Profit per Item

For every item, we compute net profit per batch and the time to make one, tracing its full ingredient chain back to whatever facilities produce the raw materials.

\[ \text{profit}_{\text{batch}} = (\text{yield} \times \text{sell_price}) - \text{raw_cost} \]

Dividing by batch time gives a per-second rate, but that rate alone doesn't say how much of each facility one batch actually needs. For that we track utilization: batches per second at this item's own facility, plus the same for every facility touched anywhere upstream in its ingredient tree (e.g. soy_sauce_tofu pulls from both Bouncy Brew Keg and Carousel Mill, which both draw soybean from Farmland).

2. Joint Facility Allocation

Every owned facility can be shared by several candidate items at once, so picking each item's rate independently would double-count shared capacity (two recipes both claiming the same Farmland soybean plots as if the other didn't exist). Instead we solve one linear program across every candidate item and every owned facility simultaneously:

\[ \max \sum_i \text{profit}_{\text{batch},i} \cdot x_i \quad \text{s.t.} \quad \sum_i \text{utilization}_{i,f} \cdot x_i \leq \text{capacity}_f \ \ \forall f \]

Where \(x_i\) is item \(i\)'s batches/sec and \(f\) ranges over every owned facility. This is solved exactly (not greedily) with an off-the-shelf simplex solver, so the result is the provably-best combination of rates achievable given shared upstream supply.

3. Rounding to Whole Units: Growers

The linear program's solution is continuous, e.g. "62% of Farmland's soybean plots." A grower facility (Farmland, Woodland, Mineral Pile, and similar) can't actually run a fractional plot; each unit commits to one crop for its whole cycle. So every grower facility's fractional shares get converted to whole counts using the largest-remainder method (the same apportionment technique used to allocate parliament seats), landing as close to the true fractions as an integer solution allows.

Each item's final rate is then capped by what its grower facilities can actually supply once rounded, taking the minimum against the continuous rate rather than recomputing from scratch, since the continuous solve already accounted for fair sharing at every processor a chain passes through.

4. Whole-Unit Dedication: Processors

A processor facility (Carousel Mill, Claw Game Cooker, and similar) isn't tied to one crop for a whole cycle the way a grower is, but it still can't be time-shared between two different recipes in practice. A player sets one up to run one recipe continuously, not a "38% of the time" split. When the linear program's solution would ask a processor facility to serve more distinct recipes than it has units, we keep only the most profitable candidates (one dedicated unit each) and drop the rest, then re-solve so their freed-up upstream supply finds a genuine next-best use rather than being silently discarded.

5. Time to Reach a Goal

Once the plan's rates are settled, each item contributes nothing until its own lead time has passed (time for its first batch to clear the chain), then its steady rate after that:

\[ \text{amount}(t) = \sum_i \text{rate}_i \cdot \max(0,\ t - \text{lead}_i) \]

This is monotonically non-decreasing in \(t\), so the smallest \(t\) reaching your target amount is found with a binary search rather than solving anything analytically.

How to Use

1. Pick a Target

Choose Coins or Bud Tickets — only recipes that sell for that currency are considered. Wood Blocks/Mineral Sand aren't a separate target; they come as a byproduct of certain Woodland/Mineral Pile crops regardless of what you're optimizing for, and the "Prioritize Wood Blocks/Mineral Sand" checkbox can guarantee their maximum possible rate even at the cost of Coins/Bud Ticket profit.

2. Configure Facilities

Set the count and level of each facility you have unlocked in Aniimo Homeland.

3. Set Upgrade Modules

Enter the level of each upgrade module you've unlocked (0 = not unlocked).

4. Calculate

Click "Calculate Optimal Path" to see your best achievable rate and what each facility should produce. No target amount is needed for this step.

5. Set a Goal (Optional)

Once you have a plan, enter a target amount and your current balance further down to see how long it will take. This updates instantly as you type, with no need to recalculate the plan.

Understanding Results

  • Your Rate: The best steady-state rate of your chosen target per second achievable with your current facilities
  • What Each Facility Should Do: Which item to dedicate each owned facility to, running in parallel
  • Total Time / Amount Produced: How long your goal takes, and the item-level breakdown of what gets made along the way

How It Decides

Every owned facility is dedicated to its own best item for the chosen target, all running simultaneously. Different facility types don't compete for the same time slot, so using more of them always finishes faster. The reported rate is the fastest achievable given your current facility counts and levels.
